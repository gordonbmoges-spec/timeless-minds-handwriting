# iPad Safari 封面连续性修复提示词

## 目标

修复“会回应的藏书阁”在真实 iPad Safari 中打开书籍时出现的两次封面消失，同时保持桌面浏览器的现有视觉效果和动画时长。

## 已确认现象

1. 用户点击书架封面后，书架封面先消失，约 0.8–1 秒后放大封面才出现。
2. 封面开始向左翻开并经过约 90°时，封面再次消失；本应接续显示的内封面没有被 Safari 绘制。
3. 两次消失的主体都是封面，不是羊皮纸，也不是整本书。
4. 同一套代码在桌面 Chromium 中基本连续，问题主要出现在 iPad Safari/WebKit。

## 根因假设

- 第一处是绘制交接竞态：原书架封面被同步设为透明，但新建的 fixed portal 图片尚未完成解码、栅格化和首帧绘制。
- 第二处是 WebKit 3D 合成问题：封面父层同时执行 `rotateY` 和动态 `filter: drop-shadow()`，造成 3D 子面被压平；正面经过 90°被背面裁剪后，内封面也未能接替显示。

## 实现要求

1. 点击后先创建 portal，等待其中所有图片 `decode()` 完成，再等待至少两个 `requestAnimationFrame`。
2. portal 已经实际进入可绘制状态后才隐藏原书架封面，并在下一帧启动位移动画。
3. 图片解码或 `decode()` 不可用时要安全降级，不能阻塞打开流程。
4. 封面翻转父层只能动画 `transform`，不得在同一层动画 `filter`。
5. 封面父层必须保持 `transform-style: preserve-3d` 且允许正反面容器存在；`backface-visibility: hidden` 只应用于正面和内面两个独立子面。
6. 为 WebKit 补充 `-webkit-transform-style` 和 `-webkit-backface-visibility`，并给两个面极小的 Z 轴分离，避免 90°附近重叠闪烁。
7. 阴影改为子面的静态 `box-shadow`，不得使用会压平 3D 上下文的动态滤镜。
8. 不改人物、书页、书架布局、回复逻辑、记忆功能和 API 行为。
9. 本轮只修改本地代码，不部署公网、不推送 GitHub。

## 验收标准

- 从点击到封面开始放大，画面中始终至少有一张完整封面可见。
- 放大封面与 3D 封面交接时无空白帧、无突然闪现。
- 封面从 0°翻到 180°全过程连续；90°前看到外封面，90°后立即看到内封面。
- 动画期间不得通过 `opacity: 0`、`visibility: hidden` 或父层背面裁剪让封面整体消失。
- 桌面浏览器测试、静态检查和完整测试套件全部通过。
- 生成逐帧接触图检查两个交接区间；真实 iPad Safari 最终仍需真机复验。
